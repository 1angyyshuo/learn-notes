# Johnny Decimal 分类法

## 概述

**Johnny Decimal** 是一种数字编码分类系统，通过 **10 个大类 × 10 个小类** 的结构，让文件和笔记的组织既有序又灵活。

## 核心规则

### 结构层次
```
10-19 大类 A    ← 编号范围定义类别
  11   小类 A1  ← 具体项目
  12   小类 A2

20-29 大类 B
  21   小类 B1
  22   小类 B2
```

### 编号规则

1. **两位数编码**：`10`, `20`, `30`... 表示大类（最多 10 个）
2. **三位数编码**：`11`, `12`, `21`... 表示小类（每个大类最多 10 个）
3. **留有余地**：每个大类预留空位用于扩展

## 我的笔记分类

```
10-Projects/      # 项目（有明确开始和结束）
  11-Active/      # 进行中项目
  12-Archived/    # 已完成项目

20-Areas/         # 领域（持续维护）
  21-Learning/    # 学习主题
  22-Work/        # 工作相关

30-Resources/     # 资源（参考资料）
  31-Books/       # 读书笔记
  32-Courses/     # 课程笔记
  33-Articles/    # 文章收藏

40-Archive/       # 归档（不再活跃但有价值）
  41-Templates/   # 笔记模板

50-Daily/         # 日常（时间敏感）
  51-Journal/     # 每日日志

60-System/        # 系统（配置和工具）
```

## 为什么这样分类？

### 优势

1. **顺序确定**：编号决定排序，不受文件名影响
2. **空间限制**：最多 100 个分类，迫使你精简
3. **灵活扩展**：预留编号方便未来添加
4. **跨平台一致**：任何文件系统都适用

### 与双链笔记的配合

- **Johnny Decimal**：负责笔记的"**存放位置**"
- **双链**：负责笔记的"**逻辑关联**"
- **结合**：既有清晰结构，又有灵活连接

## 使用建议

### 选择类别的原则

1. **问自己**：这份笔记最可能在什么情境下被找到？
2. **避免过细**：开始宁可粗一些，后续再细分
3. **保持一致**：一旦确定分类，坚持执行

### 文件命名

```
20-Areas/21-Learning/
  ├── javascript-基础.md
  ├── javascript-闭包.md
  ├── python-入门.md
  └── 算法-排序.md
```
