# 大模型推理与训练优化学习规划

## 学习目标

掌握大模型推理和训练阶段的核心优化技术，能够回答面试中的相关问题，并具备实际动手能力。

## 四个核心主题

### 1. 稀疏注意力（Sparse Attention）

**理论基础**
- 标准 Attention 的计算复杂度 O(n²)
- 稀疏性的动机：并非所有 token 对都重要
- 各类稀疏模式：局部、步幅、随机、块稀疏

**经典方法**
- Sparse Transformer（OpenAI）
- Longformer
- BigBird
- Quest（Query-aware）

**面试重点**
- 为什么需要稀疏注意力？
- 不同稀疏模式的优缺点？
- 稀疏注意力的信息损失如何保证？

**实践目标**
- 理解 Quest 代码实现
- 能画出不同稀疏模式图

---

### 2. FlashAttention

**理论基础**
- GPU 内存层次结构：HBM vs SRAM
- 标准 Attention 的 IO 瓶颈（非计算瓶颈）
- Tiling 和重计算思想

**三代演进**
- FlashAttention-1：减少 HBM 读写
- FlashAttention-2：减少非矩阵乘法运算
- FlashAttention-3：异步重叠计算（Hopper）

**面试重点**
- FlashAttention 解决了什么问题？
- 为什么 FlashAttention 节省显存但不减少计算量？
- Tiling 的具体策略？

**实践目标**
- 使用 FlashAttention 加速推理
- 理解 CUDA Kernel 优化思路

---

### 3. vLLM

**理论基础**
- KV Cache 显存管理问题
- PagedAttention：借鉴 OS 虚拟内存
- 连续批处理（Continuous Batching）

**核心机制**
- Block Table 映射
- Copy-on-Write 共享
- Prefix Caching

**面试重点**
- PagedAttention  vs  传统连续内存分配？
- vLLM 为什么能提高吞吐量？
- 内部碎片和外部碎片的区别？

**实践目标**
- 部署 vLLM 服务
- 调优 gpu_memory_utilization 等参数
- 与 HuggingFace 速度对比

---

### 4. DeepSpeed

**理论基础**
- 数据并行 vs 模型并行 vs 流水线并行
- 显存瓶颈：Optimizer States + Gradients + Parameters
- ZeRO（Zero Redundancy Optimizer）思想

**ZeRO 三阶段**
- ZeRO-1：Optimizer States 分片
- ZeRO-2：Gradients 分片
- ZeRO-3：Parameters 分片
- ZeRO-Infinity： offload 到 CPU/NVMe

**面试重点**
- ZeRO 每个阶段节省多少显存？
- 通信开销如何变化？
- ZeRO vs Megatron-LM vs FSDP？

**实践目标**
- 使用 DeepSpeed 训练 7B 模型
- 配置 ds_config.json
- 观察显存和通信变化

---

## 学习顺序

推荐按照 **FlashAttention → vLLM → 稀疏注意力 → DeepSpeed** 的顺序学习：

1. **FlashAttention** 是基础，理解 GPU 内存和计算优化
2. **vLLM** 建立在 FlashAttention 之上，解决服务层问题
3. **稀疏注意力** 是更高级的优化方向
4. **DeepSpeed** 转向训练侧，与推理优化形成完整闭环
