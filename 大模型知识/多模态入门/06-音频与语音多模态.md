## 音频信号基础

### 从声波到特征

```
声波（空气振动）
    ↓ 采样（如 16kHz）
离散信号（数字序列）
    ↓ 窗口化 + FFT
频谱图（Spectrogram）
    ↓ Mel 滤波器
Mel Spectrogram（梅尔频谱图）
```

### Mel Spectrogram

Mel Spectrogram 是音频处理的标准特征表示：

```python
import torchaudio

def extract_mel_spectrogram(waveform, sr=16000, n_mels=80):
    """
    waveform: (1, num_samples)
    返回: (1, n_mels, time_steps)
    """
    mel_transform = torchaudio.transforms.MelSpectrogram(
        sample_rate=sr,
        n_fft=400,
        hop_length=160,
        n_mels=n_mels
    )
    mel_spec = mel_transform(waveform)
    # 取对数
    mel_spec = torch.log(mel_spec + 1e-9)
    return mel_spec
```

**关键参数：**
- **采样率 (sample_rate):** 16kHz（语音）或 44.1kHz（音乐）
- **n_fft:** FFT 窗口大小，决定频率分辨率
- **hop_length:** 帧移，决定时间分辨率
- **n_mels:** Mel 滤波器数量，决定特征维度

## 音频编码器

### Whisper Encoder (OpenAI)

Whisper 是目前最流行的语音识别模型，其编码器可以作为通用音频特征提取器。

```
音频 → Mel Spectrogram (80 mel bins, 30s@16kHz = 3000 frames)
    ↓
Conv1D + GELU (下采样)
    ↓
位置编码
    ↓
Transformer Encoder (6/12/24/32 层)
    ↓
音频特征向量
```

**Whisper 编码器架构：**

```python
class WhisperEncoder(nn.Module):
    def __init__(self, n_mels=80, d_model=1280, n_layers=32):
        super().__init__()
        # 卷积下采样
        self.conv1 = nn.Conv1d(n_mels, d_model, kernel_size=3, padding=1)
        self.conv2 = nn.Conv1d(d_model, d_model, kernel_size=3, stride=2, padding=1)
        # Transformer
        self.layers = nn.ModuleList([
            TransformerEncoderLayer(d_model) for _ in range(n_layers)
        ])
        # 位置编码
        self.positional_embedding = nn.Embedding(1500, d_model)
    
    def forward(self, mel_spectrogram):
        """
        mel_spectrogram: (batch, 80, 3000)
        输出: (batch, 1500, 1280)
        """
        # 卷积下采样 2x
        x = F.gelu(self.conv1(mel_spectrogram))
        x = F.gelu(self.conv2(x))  # (batch, 1280, 1500)
        x = x.permute(0, 2, 1)  # (batch, 1500, 1280)
        
        # 加位置编码
        x = x + self.positional_embedding.weight
        
        # Transformer 编码
        for layer in self.layers:
            x = layer(x)
        
        return x  # (batch, 1500, 1280)
```

### HuBERT (Meta)

HuBERT 使用自监督学习，不需要标注数据：

```
音频 → Mel Spectrogram
    ↓
K-means 聚类 → 伪标签
    ↓
Transformer 预测伪标签
    ↓
学到的音频表示
```

**核心思想：**
- 类似 BERT 的 MLM，但用于音频
- 先用聚类生成伪标签，然后预测被 mask 的部分

### Wav2Vec 2.0 (Meta)

```
音频 → CNN 特征提取
    ↓
对比学习（正样本 vs 负样本）
    ↓
量化模块
    ↓
Transformer 编码
```

## 语音 Token 化方法

### 问题

LLM 处理的是离散 token，但音频是连续信号。需要将音频转换为 token。

### 方法一：连续 Token（Continuous Tokens）

直接用音频特征作为 LLM 的输入：

```python
# Whisper 编码 → 连续特征 → 直接输入 LLM
audio_features = whisper_encoder(audio)  # (batch, 1500, 1280)
projected = projection_layer(audio_features)  # (batch, 1500, 4096)
llm_input = torch.cat([projected, text_tokens], dim=1)
```

**优点：** 信息完整
**缺点：** Token 数量大，与 LLM 离散输入不一致

### 方法二：离散 Token（Discrete Tokens）

用 VQ-VAE 或类似方法将音频量化为离散 token：

```python
class AudioTokenizer(nn.Module):
    def __init__(self, codebook_size=8192, d_model=128):
        super().__init__()
        self.encoder = AudioEncoder()
        self.codebook = nn.Embedding(codebook_size, d_model)
        self.decoder = AudioDecoder()
    
    def encode(self, audio):
        """音频 → 离散 token 序列"""
        features = self.encoder(audio)  # (batch, T, d_model)
        # 找到最近的 codebook 向量
        distances = torch.cdist(features, self.codebook.weight)
        tokens = distances.argmin(dim=-1)  # (batch, T)
        return tokens
    
    def decode(self, tokens):
        """离散 token → 音频"""
        features = self.codebook(tokens)  # (batch, T, d_model)
        audio = self.decoder(features)
        return audio
```

**优点：** 可以用标准 LLM 架构
**缺点：** 量化损失

### 方法三：混合方案

同时使用连续和离散 token：

```
音频 → 连续特征（用于编码器）
    ↓
VQ 量化 → 离散 token（用于 LLM 生成）
```

## 多模态语音模型

### Qwen-Audio (阿里巴巴)

**架构：**
```
音频 → Whisper Encoder → 投影层 → Qwen-7B
```

**特点：**
- 支持 30+ 种语言
- 统一处理语音识别、翻译、理解
- 支持音频问答（Audio QA）

### Salmonn (多机构联合)

**架构：**
```
音频 → Whisper Encoder ─┐
                       ├→ Q-Former → Vicuna-13B
音频 → BEATs Encoder ──┘
```

**特点：**
- 双编码器：Whisper（语音）+ BEATs（一般音频/音乐）
- Q-Former 对齐
- 支持语音、音乐、环境音理解

### AnyGPT (Shanghai AI Lab)

**架构：**
```
任何模态 → Token 化 → 统一 LLM → 任何模态
```

**特点：**
- 统一的 token 化方案
- 文本、图像、音频、音乐都转换为离散 token
- 用同一个 LLM 处理

### SpeechGPT (CUHK)

**架构：**
```
音频 → HuBERT → 离散 token → LLM → 离散 token → HiFi-GAN → 音频
```

**特点：**
- 语音对话能力
- 用离散 token 实现端到端语音交互

## 语音对话模型

### 技术挑战

| 挑战 | 说明 |
|------|------|
| **实时性** | 用户期望低延迟响应（< 500ms） |
| **流式处理** | 不能等用户说完才开始处理 |
| **打断** | 用户可能随时打断模型 |
| **情感** | 需要理解语气、情感 |
| **多语言** | 可能混合多种语言 |

### 实时语音交互架构

```
用户语音输入
    ↓
VAD（语音活动检测）→ 检测说话开始/结束
    ↓
ASR（语音识别）→ 文本
    ↓
LLM → 文本回复
    ↓
TTS（文本转语音）→ 语音输出
    ↓
扬声器播放
```

### 端到端方案

```
音频输入 → 编码器 → 离散 token → LLM → 离散 token → 解码器 → 音频输出
```

**代表模型：** GPT-4o、Gemini、Qwen-Audio

## 音乐理解与生成

### 音乐特征

| 特征 | 说明 | 提取方法 |
|------|------|---------|
| **节奏** | BPM（每分钟节拍数） | 节拍检测 |
| **音高** | 音符的频率 | 频谱分析 |
| **和弦** | 同时发声的音符组合 | 和弦识别 |
| **音色** | 乐器的特征 | 音色分析 |
| **结构** | 前奏、主歌、副歌等 | 结构分析 |

### 音乐模型

| 模型 | 功能 | 特点 |
|------|------|------|
| **MusicGen** | 文本→音乐 | Meta 开源 |
| **Jukebox** | 音乐生成 | OpenAI，支持歌词 |
| **MusicLM** | 文本→音乐 | Google |
| **Suno** | 文本→歌曲 | 商业产品 |

### 音乐多模态理解

```python
# 用音频编码器提取音乐特征
music_features = audio_encoder(music)  # (batch, T, d_model)

# 用 LLM 理解音乐
input_tokens = torch.cat([music_features, text_tokens], dim=1)
output = llm(input_tokens)
# → "这首歌曲是爵士风格，BPM 120，主要使用萨克斯和钢琴"
```
