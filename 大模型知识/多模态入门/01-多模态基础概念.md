# 01 - 多模态基础概念

## 什么是多模态学习？

**多模态学习（Multimodal Learning）** 是机器学习的一个分支，指模型同时处理和整合**多种模态（Modality）**的数据，模仿人类通过视觉、听觉、语言等多种感官联合认知世界的方式。

### 核心模态类型

| 模态 | 英文 | 数据类型 | 典型应用 |
|------|------|---------|---------|
| **文本** | Text | 自然语言序列 | 翻译、摘要、问答 |
| **图像** | Image | 像素矩阵（2D/3D） | 分类、检测、分割 |
| **视频** | Video | 连续图像帧 + 时序 | 行为识别、视频理解 |
| **音频** | Audio | 波形 / 频谱 | 语音识别、音乐理解 |
| **3D 点云** | 3D Point Cloud | $(x,y,z)$ 坐标集合 | 自动驾驶、机器人 |
| **触觉/力学** | Tactile / Force | 传感器时序信号 | 具身智能 |
| **表格** | Tabular | 结构化字段 | 数据分析 |
| **代码** | Code | 结构化文本 | 代码生成与理解 |

**核心目标：** 打破模态之间的语义鸿沟（Semantic Gap），将异构数据统一到共享语义空间，实现**跨模态的理解与生成**。

## 三个核心阶段

多模态学习的完整流程可抽象为三个阶段：**Feature Extraction → Fusion → Generation**。

```
                      ┌─────────────────────────────────┐
                      │        Complete Pipeline         │
                      │                                 │
  ┌─────────┐        ┌─────────┐        ┌─────────────┐│
  │ Image   │───┐    │         │    ┌──►│ Image Caption││
  └─────────┘   ▼    │  Fusion │    │   └─────────────┘│
              ┌──────┐│  Space  │┌───┴───┐             │
              │Feat. ││   ──►   ││Decode │──► Text     ││
              │Ext.  ││         ││       │             ││
              └──────┘│         │└───┬───┘             ││
  ┌─────────┐   ▲    │         │    │   ┌─────────────┐│
  │ Text    │───┘    └─────────┘    └──►│ Image Gen.   ││
  └─────────┘                          └─────────────┘│
                      └─────────────────────────────────┘
```

### 阶段一：Feature Extraction（特征提取）

将原始数据（像素、波形、文本序列）转换为**稠密向量表示（Dense Embedding）**。

| 模态 | 典型编码器 | 输出形式 |
|------|-----------|---------|
| 图像 | **ViT**、**ResNet**、**ConvNeXt** | Patch tokens / Feature map |
| 文本 | **BERT**、**LLaMA**、**T5** | Token embeddings |
| 音频 | **Whisper**、**HuBERT**、**CLAP** | Frame-level features |
| 视频 | **VideoMAE**、**TimeSformer** | Spatiotemporal tokens |
| 3D | **Point-BERT**、**3D-ViT** | Point/voxel tokens |

**关键挑战：**
- 不同模态的**特征维度**、**分布**、**语义粒度**完全不同
- 图像特征是空间连续的，文本特征是离散符号的
- 时序模态（视频、音频）需要额外建模时间维度

### 阶段二：Fusion（模态融合）

将来自不同模态的特征整合为**统一的多模态表示**，是整个流程的核心。

核心要解决三个问题：
- **何时融合**（When）：早期 / 中期 / 晚期
- **如何融合**（How）：拼接 / 加法 / 注意力 / 门控
- **在哪融合**（Where）：单层 / 逐层 / 多尺度

### 阶段三：Generation（生成）

基于融合后的多模态理解，生成目标模态的输出。

| 任务方向 | 典型任务 | 代表模型 |
|---------|---------|---------|
| 图像 → 文本 | Image Captioning、VQA、OCR | GPT-4V、Qwen-VL |
| 文本 → 图像 | Text-to-Image Generation | Stable Diffusion、DALL-E 3 |
| 文本 → 音频 | Text-to-Speech (TTS) | VALL-E、Bark |
| 图像 + 文本 → 代码 | UI-to-Code | GPT-4o |
| 多模态 → 动作 | Robot Manipulation | RT-2、Octo |

## 融合方式对比

### 核心融合范式

```
(1) Early Fusion            (2) Intermediate Fusion       (3) Late Fusion
    原始信号级                   中间特征级                    高层语义级

  Img  Txt  Aud            Img ───── Txt ───── Aud       Img ──── Txt ──── Aud
   │    │    │               │         │         │         │        │        │
   └────┼────┘               └────┬────┘         │         ▼        ▼        ▼
        ▼                        ▼               │      Encoder  Encoder  Encoder
    拼接/加法              Cross-Attn / Gated     │         │        │        │
        │                    ┌──┴──┐              │         └────┬───┘        │
        ▼                    │Fused│◄─────────────┘              ▼            │
    Joint Encoder            └──┬──┘                        Score / Dot ─────┘
        │                       ▼
        ▼                    Joint Encoder
    下游任务                      │
                                ▼
                            下游任务
```

### 详细对比

| 融合方式 | 原理 | 融合位置 | 优点 | 缺点 | 代表模型 |
|---------|------|---------|------|------|---------|
| **早期融合 (Early Fusion)** | 原始特征直接拼接（Concat）或加和（Add） | 输入层 | 简单直观，保留原始信息 | 特征未对齐，计算量大 | VisualBERT、ViLT |
| **中期融合 (Intermediate Fusion)** | 通过交叉注意力或门控在 Transformer 层间交换信息 | 中间层 | 深度交互，效果最优 | 计算复杂度 $O(n^2)$ | Flamingo、LLaVA、Qwen-VL |
| **晚期融合 (Late Fusion)** | 各模态独立编码，仅在最终决策层融合 | 输出层 | 模块化，易扩展，计算高效 | 缺乏细粒度交互 | CLIP、SigLIP |
| **层级融合 (Hierarchical Fusion)** | 在多个粒度层级同时融合（如 ResNet 各 stage） | 多层 | 多尺度信息完整 | 结构复杂 | VATT、ViLT |
| **自适应融合 (Adaptive Fusion)** | 动态学习各模态的重要性权重 | 任意位置 | 可处理模态缺失 | 训练复杂 | MMTM、G-Blend |

## 应用场景

### 1. 视觉问答（VQA — Visual Question Answering）
- **任务：** 给定图像和自然语言问题，生成正确答案
- **技术链：** 视觉编码 → 文本编码 → 跨模态注意力 → 答案解码
- **核心挑战：** 空间推理、计数、OCR、常识推理
- **代表基准：** VQA v2、GQA、TextVQA、MMBench

### 2. 图像描述与视觉叙事（Image Captioning & Visual Storytelling）
- **任务：** 为图像生成自然语言描述，或为图像序列生成连贯故事
- **技术链：** 视觉编码 → 视觉-语言对齐 → 自回归文本生成
- **核心挑战：** 细粒度属性识别、关系推理、多样性

### 3. 光学字符识别（OCR）与文档理解
- **任务：** 从图像中提取文字，理解文档布局
- **技术链：** 文本检测 → 文本识别 → 布局分析 → 信息抽取
- **核心挑战：** 复杂版式、手写体、公式、多语言混合

### 4. 医疗多模态（Medical Multimodal）
- **任务：** 影像诊断、报告生成、病理分析
- **模态组合：** X 光 / CT / MRI + 病历文本 + 基因组学
- **核心挑战：** 数据稀缺与隐私、专业标注成本、高风险决策

### 5. 自动驾驶（Autonomous Driving）
- **任务：** 环境感知、轨迹预测、端到端规划
- **模态组合：** RGB 相机 + LiDAR + RADAR + 高精地图 + GPS
- **核心挑战：** 实时性（毫秒级）、安全关键、极端天气鲁棒性

### 6. 具身智能（Embodied AI）
- **任务：** 机器人操作、移动导航、人机协作
- **模态组合：** 视觉 + 语言指令 + 触觉 + 本体感知
- **核心挑战：** Sim-to-Real 迁移、长程任务规划、安全约束

### 7. 视频理解（Video Understanding）
- **任务：** 行为识别、视频问答、时序定位、视频摘要
- **技术链：** 帧采样 → 时序编码 → 多模态融合 → 理解/生成
- **核心挑战：** 长视频依赖、计算开销、时空联合建模

### 8. 代码与 UI 多模态（Code & UI Multimodal）
- **任务：** UI 截图转代码、代码审查、架构图理解
- **模态组合：** 截屏 + 代码 + 自然语言
- **核心挑战：** 长上下文、精确像素级对应

## 关键术语表

| # | 术语（中文） | 英文 | 解释 |
|---|-----------|------|------|
| 1 | **模态** | Modality | 数据的感知或表示形式，如图像、文本、音频 |
| 2 | **多模态学习** | Multimodal Learning | 让模型同时处理和关联多种模态数据的机器学习范式 |
| 3 | **模态对齐** | Modality Alignment | 将不同模态的特征映射到**统一语义空间**的技术 |
| 4 | **语义鸿沟** | Semantic Gap | 不同模态的低级特征与高级语义之间的差异 |
| 5 | **跨模态注意力** | Cross-Modal Attention | 一种模态作为 Query，另一种模态提供 Key/Value 的注意力机制 |
| 6 | **对比学习** | Contrastive Learning | 拉近正样本对、推远负样本对，学习共享表示 |
| 7 | **视觉编码器** | Vision Encoder | 将图像转化为特征向量的神经网络，如 ViT |
| 8 | **投影层/适配器** | Projection Layer / Adapter | 连接视觉编码器与语言模型的**线性或 MLP 映射层** |
| 9 | **Q-Former** | Querying Transformer | BLIP-2 提出的使用可学习 Query token 聚合视觉特征的模块 |
| 10 | **视觉问答** | VQA (Visual Question Answering) | 给定图像和自然语言问题，输出答案 |
| 11 | **视觉定位** | Visual Grounding | 根据文本描述找到图像中对应的**空间区域** |
| 12 | **幻觉** | Hallucination | 模型生成的内容与输入图像**事实不符**的现象 |
| 13 | **指令微调** | Instruction Tuning | 使用 (指令, 回答) 对微调模型，使其更好地遵循人类意图 |
| 14 | **零样本学习** | Zero-shot Learning | 在**未见过的任务/类别**上直接推理，无需额外训练 |
| 15 | **动态分辨率** | Dynamic Resolution | 允许模型处理**不同尺寸和宽高比**的输入图像 |
| 16 | **图像分片** | Image Tiling / Patchify | 将大图切分为固定大小的 Patch 后分别编码 |
| 17 | **Token 化** | Tokenization | 将连续数据（如图像 Patch）映射为**离散或连续 Token** 的过程 |
| 18 | **视觉 Token** | Visual Token | 图像 Patch 经编码器输出的向量表示，输入 LLM 的"词" |
| 19 | **特征对齐** | Feature Alignment | 不同模态的特征在**分布、维度、语义**上保持一致 |
| 20 | **模态缺失** | Missing Modality | 训练或推理时某些模态数据不可用的情况，需鲁棒处理 |
| 21 | **原生多模态** | Native Multimodality | 模型从设计之初就统一处理多模态输入（如 Gemini），而非拼接方案 |
| 22 | **多模态 MoE** | Multimodal Mixture of Experts | 使用不同专家子网络处理不同模态/子任务，提高效率 |
| 23 | **端到端训练** | End-to-End Training | 所有模块（编码-融合-解码）联合优化，无需分阶段 |
| 24 | **扩散模型** | Diffusion Model | 通过逐步去噪生成高质量图像/音频的生成范式 |
| 25 | **自回归生成** | Autoregressive Generation | 逐 token 预测下一个输出的生成方式（如 GPT 系列） |

## 大模型时代多模态架构范式

```
                   ┌────────────────────────────────┐
                   │    主流 MLLM 架构（2023-2026）    │
                   │                                │
  ┌──────────┐    ┌───────────┐    ┌──────────────┐│
  │ Vision   │───►│ Projector │───►│ LLM          ││
  │ Encoder  │    │           │    │              ││
  │ (ViT/CNN)│    │ MLP /     │    │ (LLaMA/Qwen/ ││
  │          │    │ Q-Former/ │    │  DeepSeek)   ││
  │          │    │ Resampler │    │              ││
  └──────────┘    └───────────┘    └──────┬───────┘│
                                          │        │
      图像/视频输入                   ┌────▼────┐   │
                                    │ 文本输出  │   │
                                    └─────────┘   │
                   └────────────────────────────────┘
```

**三件套公式：** `Vision Encoder + Projector + LLM = MLLM`

这个简洁架构已成为 2023 年至今几乎所有多模态大模型的**事实标准**，变体主要体现在：
- 视觉编码器的选择（ViT-L、SigLIP、InternViT 等）
- 投影层的设计（简单 MLP vs Q-Former vs Perceiver Resampler）
- 分辨率策略（固定分辨率 vs 动态分片）
