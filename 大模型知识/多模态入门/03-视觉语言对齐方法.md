# 03 - 视觉语言对齐方法

## 为什么需要对齐？

视觉编码器（如 ViT）和语言模型（如 LLM）在完全不同的空间中训练：
- **视觉特征：** 来自图像像素，编码的是颜色、形状、空间关系
- **文本嵌入：** 来自语言 token，编码的是语义、语法

要让 LLM 能"看懂"图像，必须将视觉特征**对齐**到文本嵌入空间。

```
视觉空间                    对齐层                    文本空间
┌──────────┐              ┌──────────┐              ┌──────────┐
│ [0.3, 0.8│   ──────►   │  投影层   │   ──────►   │ [0.1, 0.5│
│ -0.2, 1.1│              │ (MLP/Q-  │              │  0.3, -0.│
│  0.5, ...│              │  Former) │              │  4, 0.7..│
└──────────┘              └──────────┘              └──────────┘
 图像 patch 特征            可学习映射                token 嵌入
 (ViT 输出)                                          (LLM 输入)
```

---

## 四种主流对齐方法

### 方法一：Cross-Attention（Flamingo 风格）

**代表模型：** Flamingo (DeepMind, 2022)

#### 架构

```
视觉特征 ──► Perceiver Resampler ──► 固定数量的视觉 token
                                          │
文本 token ────────────────────────────► Gated Cross-Attention ──► 文本 token
                                              (视觉作为 Key/Value)
```

#### 核心组件

**1. Perceiver Resampler（感知重采样器）**

将变长的视觉特征压缩为固定数量的 token：

```python
class PerceiverResampler(nn.Module):
    def __init__(self, num_queries=64, hidden_dim=1024, num_heads=8):
        super().__init__()
        # 可学习的查询 token
        self.queries = nn.Parameter(torch.randn(num_queries, hidden_dim))
        # 交叉注意力层
        self.cross_attn = nn.MultiheadAttention(hidden_dim, num_heads)
    
    def forward(self, visual_features):
        """
        visual_features: (num_patches, hidden_dim)  例如 (576, 1024)
        输出: (num_queries, hidden_dim)  例如 (64, 1024)
        """
        # queries 作为 Query，视觉特征作为 Key/Value
        output = self.cross_attn(
            query=self.queries.unsqueeze(1),  # (64, 1, 1024)
            key=visual_features.unsqueeze(1),  # (576, 1, 1024)
            value=visual_features.unsqueeze(1)
        )
        return output.squeeze(1)  # (64, 1024)
```

**2. Gated Cross-Attention（门控交叉注意力）**

在 LLM 的每个 Transformer 层中插入交叉注意力：

```python
class GatedCrossAttention(nn.Module):
    def __init__(self, hidden_dim):
        super().__init__()
        self.cross_attn = nn.MultiheadAttention(hidden_dim, num_heads=8)
        # 门控机制：初始值为 0，让模型从纯文本能力开始
        self.gate = nn.Parameter(torch.zeros(1))
    
    def forward(self, text_features, visual_features):
        # 交叉注意力
        attn_out = self.cross_attn(
            query=text_features,
            key=visual_features,
            value=visual_features
        )
        # 门控残差连接
        # tanh(gate) 初始为 0，训练初期不影响文本能力
        return text_features + torch.tanh(self.gate) * attn_out
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 深度交互，效果好 | 计算复杂度高 |
| 不改变原始 LLM 结构 | 需要修改 LLM 每一层 |
| 少样本学习能力强 | 训练成本高 |
| 可处理高分辨率图像 | 推理速度较慢 |

### 方法二：Visual Token（LLaVA 风格）

**代表模型：** LLaVA, LLaVA-1.5, Qwen-VL

#### 架构

```
图像 ──► ViT ──► 视觉特征 ──► MLP Projection ──► 视觉 Token
                                                    │
文本 ──► Tokenizer ──► 文本 Token ────────────────► LLM ──► 回答
                                                    │
                                         [视觉Token + 文本Token]
```

#### 核心组件

**MLP Projection（多层感知机投影）**

最简单的对齐方式——用一个 MLP 将视觉特征映射到文本空间：

```python
class MLPProjection(nn.Module):
    def __init__(self, vision_dim=1024, text_dim=4096):
        super().__init__()
        self.proj = nn.Sequential(
            nn.Linear(vision_dim, text_dim),
            nn.GELU(),
            nn.Linear(text_dim, text_dim)
        )
    
    def forward(self, visual_features):
        """
        visual_features: (batch, num_patches, vision_dim)  (1, 576, 1024)
        输出: (batch, num_patches, text_dim)  (1, 576, 4096)
        """
        return self.proj(visual_features)
```

#### LLaVA-1.5 的完整流程

```python
# 1. 图像编码
image = preprocess(image)  # (224, 224) → (3, 224, 224)
visual_features = vit(image)  # (1, 257, 1024)  [CLS + 256 patches]
visual_features = visual_features[:, 1:, :]  # 去掉CLS → (1, 256, 1024)

# 2. 投影到文本空间
visual_tokens = mlp_projection(visual_features)  # (1, 256, 4096)

# 3. 拼接文本 token
text_tokens = tokenizer("描述这张图片")  # (1, 6, 4096)
input_tokens = torch.cat([visual_tokens, text_tokens], dim=1)  # (1, 262, 4096)

# 4. LLM 生成
output = llm(input_tokens)  # → "这是一只可爱的猫咪在沙发上睡觉"
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单，只需训练投影层 | 视觉 token 数量多，计算量大 |
| 不修改 LLM 结构 | 缺乏深度跨模态交互 |
| 训练成本低 | 对高分辨率图像处理较弱 |
| 效果出乎意料地好 | 幻觉问题较严重 |

### 方法三：Q-Former（BLIP-2 风格）

**代表模型：** BLIP-2, InstructBLIP

#### 架构

```
图像 ──► ViT ──► 视觉特征 ──► Q-Former ──► 压缩后的视觉 Token
                         ↑                      │
                    交叉注意力              LLM ──► 回答
                         ↑                      │
               可学习查询 Token              [Token + 文本]
```

#### Q-Former 结构

Q-Former 是一个小型 Transformer，包含：
- **自注意力层：** 查询 token 之间交互
- **交叉注意力层：** 查询 token 从视觉特征中提取信息
- **FFN 层：** 特征变换

```python
class QFormerBlock(nn.Module):
    def __init__(self, hidden_dim=768, num_heads=12):
        super().__init__()
        # 自注意力（查询之间交互）
        self.self_attn = nn.MultiheadAttention(hidden_dim, num_heads)
        # 交叉注意力（查询从视觉提取信息）
        self.cross_attn = nn.MultiheadAttention(hidden_dim, num_heads)
        # FFN
        self.ffn = nn.Sequential(
            nn.Linear(hidden_dim, hidden_dim*4),
            nn.GELU(),
            nn.Linear(hidden_dim*4, hidden_dim)
        )
        self.norm1 = nn.LayerNorm(hidden_dim)
        self.norm2 = nn.LayerNorm(hidden_dim)
        self.norm3 = nn.LayerNorm(hidden_dim)
    
    def forward(self, queries, visual_features):
        # 自注意力
        queries = queries + self.self_attn(
            self.norm1(queries), self.norm1(queries), self.norm1(queries)
        )
        # 交叉注意力
        queries = queries + self.cross_attn(
            self.norm2(queries), visual_features, visual_features
        )
        # FFN
        queries = queries + self.ffn(self.norm3(queries))
        return queries

class QFormer(nn.Module):
    def __init__(self, num_queries=32, hidden_dim=768, num_layers=6):
        super().__init__()
        self.queries = nn.Parameter(torch.randn(num_queries, hidden_dim))
        self.blocks = nn.ModuleList([QFormerBlock() for _ in range(num_layers)])
    
    def forward(self, visual_features):
        queries = self.queries.unsqueeze(1).repeat(1, visual_features.shape[1], 1)
        for block in self.blocks:
            queries = block(queries, visual_features)
        return queries  # (32, batch, 768) → 32个压缩后的视觉 token
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 有效压缩视觉信息 | Q-Former 需要额外预训练 |
| 32 个 token 代替几百个 | 压缩会丢失细节信息 |
| 减少 LLM 计算量 | 两次训练（Q-Former + LLM） |
| 适合资源受限场景 | 不如直接投影简单 |

### 方法四：Linear Projection（线性投影）

**代表模型：** 一些轻量级方案

#### 架构

最简方案——只用一个线性层：

```python
class LinearProjection(nn.Module):
    def __init__(self, vision_dim=1024, text_dim=4096):
        super().__init__()
        self.proj = nn.Linear(vision_dim, text_dim, bias=False)
    
    def forward(self, visual_features):
        return self.proj(visual_features)
```

#### 为什么简单却有效？

- 现代视觉编码器（如 CLIP ViT）已经学到了很好的语义表示
- 线性投影足以将 CLIP 特征对齐到 LLM 空间
- 复杂的投影层反而容易过拟合

## Token 压缩策略

视觉 token 数量直接影响 LLM 的计算量。压缩策略很重要：

| 策略 | 方法 | Token 数 | 信息损失 | 代表模型 |
|------|------|---------|---------|---------|
| **不压缩** | 直接用所有 patch | 256-1024+ | 无 | LLaVA-1.5 |
| **固定压缩** | Q-Former/Perceiver | 32-64 | 中等 | BLIP-2, Flamingo |
| **空间合并** | 相邻 patch 合并 | 减少 4x | 低 | InternVL |
| **注意力池化** | 学习注意力权重 | 1-64 | 中等 | CogVLM |
| **动态选择** | 根据内容选关键 patch | 变长 | 自适应 | 轻量方案 |

### 空间合并示例

```python
def spatial_merge(features, merge_size=2):
    """
    将相邻的 2x2 patches 合并为 1 个 token
    (batch, 16*16, dim) → (batch, 8*8, dim*4) → (batch, 8*8, dim)
    """
    b, n, d = features.shape
    h = w = int(n**0.5)
    features = features.reshape(b, h, w, d)
    
    # 合并相邻 patches
    merged = features.reshape(b, h//merge_size, merge_size, w//merge_size, merge_size, d)
    merged = merged.permute(0, 1, 3, 2, 4, 5).reshape(b, h//merge_size, w//merge_size, merge_size*merge_size*d)
    
    # 投影回原始维度
    merged = self.merge_proj(merged)  # (dim*4) → dim
    return merged.reshape(b, -1, d)
```

## 动态分辨率支持

### 问题

- 固定分辨率（如 224×224）限制了细节感知
- 不同宽高比的图像需要统一处理

### 解决方案

**1. 动态 Patch 数量**

根据图像尺寸动态计算 patch 数量：

```python
def dynamic_patches(image, patch_size=14, min_pixels=3136, max_pixels=1003520):
    """
    根据图像尺寸动态调整
    min_pixels: 最小 patch 数（3136 = 224*224/14/14 = 16*16 = 256 patches, 但允许更小）
    max_pixels: 最大 patch 数
    """
    h, w = image.shape[-2:]
    # 保持宽高比，调整尺寸
    aspect = w / h
    target_h = int((min_pixels / aspect) ** 0.5)
    target_w = int(target_h * aspect)
    
    # 确保是 patch_size 的整数倍
    target_h = (target_h // patch_size) * patch_size
    target_w = (target_w // patch_size) * patch_size
    
    return resize(image, (target_h, target_w))
```

**2. 2D 位置编码**

使用旋转位置编码（RoPE）的 2D 版本处理不同尺寸：

```python
def rope_2d(height, width, dim):
    """生成 2D 旋转位置编码"""
    # 水平方向
    pos_h = torch.arange(height).float()
    freqs_h = 1.0 / (10000 ** (torch.arange(0, dim, 2).float() / dim))
    angles_h = pos_h[:, None] * freqs_h[None, :]
    
    # 垂直方向
    pos_w = torch.arange(width).float()
    freqs_w = 1.0 / (10000 ** (torch.arange(0, dim, 2).float() / dim))
    angles_w = pos_w[:, None] * freqs_w[None, :]
    
    # 组合
    angles = angles_h[:, None, :] + angles_w[None, :, :]  # (H, W, dim//2)
    return torch.cat([torch.cos(angles), torch.sin(angles)], dim=-1)
```

## 各方法在实际模型中的应用

| 模型 | 对齐方法 | 视觉 Token 数 | 训练策略 |
|------|---------|-------------|---------|
| **Flamingo** | Perceiver + Cross-Attn | 64 | 两阶段（冻结 LLM） |
| **BLIP-2** | Q-Former | 32 | 两阶段（冻结 LLM） |
| **LLaVA** | MLP Projection | 257 | 两阶段 |
| **LLaVA-1.5** | MLP Projection | 576 | 两阶段 |
| **Qwen-VL** | 单层交叉注意力 | 变长 | 三阶段 |
| **Qwen2-VL** | MLP + 动态分辨率 | 变长 | 三阶段 |
| **InternVL** | MLP + 分片 | 变长 | 渐进式训练 |
| **DeepSeek-VL** | Hybrid: Q-Former + MLP | 变长 | 三阶段 |
| **GPT-4V** | 未知（推测 MLP） | 未知 | 未知 |

## 方法选择建议

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| **快速实验** | Linear/MLP Projection | 最简单，效果够用 |
| **资源受限** | Q-Former | Token 数少，推理快 |
| **高分辨率** | 动态分辨率 + MLP | 保留更多细节 |
| **少样本学习** | Cross-Attention | Flamingo 效果验证 |
| **追求最佳** | MLP + 动态分辨率 | Qwen2-VL 等验证 |
