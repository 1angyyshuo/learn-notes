# 04 - 主流多模态模型架构

## 概述

本章详细分析当前主流的多模态大模型，包括架构设计、训练方法和创新点。

---

## LLaVA 系列

### LLaVA 1.0 (2023.04)

**核心贡献：** 首次提出 Visual Instruction Tuning（视觉指令微调）

```
架构：
  CLIP ViT-L/14 → 线性投影 → Vicuna-13B
  
训练：
  Stage 1: 预训练（冻结 ViT + LLM，只训练投影层）
  Stage 2: 指令微调（ViT + 投影 + LLM 全部训练）
```

**关键创新：**
- 用 GPT-4 自动生成视觉指令数据（LLaVA-Instruct-150K）
- 证明了简单的 MLP 投影 + 指令微调就能产生强大的视觉理解能力

### LLaVA-1.5 (2023.10)

**改进：**

| 改进点 | LLaVA 1.0 | LLaVA-1.5 |
|--------|-----------|-----------|
| 视觉编码器 | CLIP ViT-L/14 (224px) | CLIP ViT-L/14 (336px) |
| 投影层 | 线性层 | 2层 MLP |
| 数据 | 150K 指令 | 665K 指令 |
| 训练 | 两阶段 | 两阶段 + 更长训练 |
| 性能 | 中等 | 开源 SOTA |

**架构细节：**
```
输入图像 336×336
    ↓
CLIP ViT-L/14 @ 336px → 576 个 patch 特征 (每个 1024 维)
    ↓
2层 MLP (1024 → 4096 → 4096)
    ↓
576 个视觉 token (每个 4096 维)
    ↓
拼接文本 token → Vicuna-13B-Chat-v1.5
    ↓
文本回答
```

### LLaVA-NeXT / LLaVA-OneVision (2024)

**核心改进：**
- **动态分辨率：** 支持多种输入分辨率（336/672/1344px）
- **图像分片：** 大图切分为多个 336×336 的小块
- **视频支持：** 扩展到视频理解

```
高分辨率图像 672×672
    ↓
分片为 2×2 = 4 个 336×336 的 tile + 1 个全局 336×336 缩略图
    ↓
每个 tile → CLIP ViT → 576 特征 → MLP
    ↓
5 组 × 576 = 2880 个视觉 token
    ↓
LLaVA-NeXT → 回答
```

## Qwen-VL 系列

### Qwen-VL (2023.08)

**架构：**
```
ViT (Qwen-ViT, 6.75B) 
    ↓
单层交叉注意力 (Cross-Attention)
    ↓
Qwen-7B-Chat
```

**关键设计：**
- **单层交叉注意力：** 在 LLM 第一层后插入一个交叉注意力层
- **三阶段训练：** 预训练 → 多任务训练 → 指令微调
- **支持中文：** 首个强大的中文多模态模型

### Qwen2-VL (2024.09)

**重大改进：**

**1. 动态分辨率 + Naive Dynamic Resolution**
```
任意尺寸图像
    ↓
根据图像尺寸动态调整 patch 数量
（保持宽高比，最小 3136 pixels，最大 1003520 pixels）
    ↓
2D RoPE 位置编码（处理不同尺寸）
    ↓
ViT 编码
    ↓
MLP 投影 → LLM
```

**2. 2D-RoPE（二维旋转位置编码）**
```python
# 1D-RoPE: 只编码序列位置
position_encoding(seq_length)

# 2D-RoPE: 分别编码高度和宽度位置
position_encoding_h(height) + position_encoding_w(width)
# 这样无论图像尺寸如何，模型都能理解空间关系
```

**3. 统一的图像和视频处理**
- 视频 = 特殊排列的多帧图像
- 每 2 帧合并为一个 token（减少 token 数量）

### Qwen2.5-VL (2025.01)

**进一步优化：**
- 更大的视觉编码器（支持 1B/3B/7B/72B）
- 改进的动态分辨率策略
- 更好的视频理解能力
- Agent 能力增强（GUI 理解、操作）

## InternVL 系列

### InternVL 1.5 (2024.04)

**架构：**
```
InternViT-6B (6B 参数的视觉编码器)
    ↓
Pixel Shuffle（2x 空间合并，减少 token 数）
    ↓
MLP 投影层
    ↓
InternLM2-20B / Vicuna-13B
```

**关键特点：**
- **超大视觉编码器：** InternViT-6B，比常见的 300M ViT 大 20 倍
- **渐进式对齐训练：** 从小到大逐步训练
- **动态高分辨率：** 支持 4K 分辨率图像

### InternVL 2 (2024.07)

**改进：**
- 更统一的架构（图像 + 视频 + 多图）
- 更好的训练数据
- 支持更多任务（文档理解、OCR、图表分析）
- 1B-105B 多种尺寸

### InternVL 训练策略

```
阶段 1: InternViT 预训练
  - 数据: 大规模图文对
  - 方法: 对比学习 + 掩码图像建模

阶段 2: MLP 投影预训练
  - 数据: 大规模图文描述数据
  - 冻结 InternViT + LLM，只训练 MLP

阶段 3: 全模型微调
  - 数据: 高质量指令数据
  - InternViT + MLP + LLM 全部训练
```

## DeepSeek-VL2 (2024.12)

**架构特点：**

**1. MoE（Mixture of Experts）在多模态中的应用**
```
视觉编码器 → 动态切片编码 → MLP 投影
                                    ↓
文本 Token ──────────────────► MoE Transformer (DeepSeek-MoE)
                                        ↓
                                    文本回答
```

**2. 动态切片（Dynamic Tiling）**
- 将高分辨率图像切分为 384×384 的 tiles
- 每个 tile 独立编码
- 加上全局缩略图

**3. 视觉-语言 MoE**
- 不是所有专家都处理视觉信息
- 有专门的视觉专家和文本专家
- 路由机制自动分配

## Kimi-VL (Moonshot, 2025)

**设计理念：** 轻量高效

**特点：**
- 使用较小的视觉编码器
- 高效的 token 压缩策略
- 支持长上下文（128K+ tokens）
- 强大的多图理解能力

**架构：**
```
视觉编码器 → 压缩模块 → MLP → Kimi 模型
```

---

## GPT-4V / GPT-4o (OpenAI, 闭源)

**已知信息（推测）：**

根据 OpenAI 发布的技术报告和分析：

1. **原生多模态：** 不是视觉编码器 + LLM 的拼接，而是端到端训练的统一模型
2. **高分辨率处理：** 支持任意分辨率图像
3. **强大的 OCR：** 能识别复杂版式文档
4. **推理能力：** 能基于图像进行复杂推理

**技术线索：**
- 可能使用了 ViT 变体作为视觉前端
- 可能采用了动态分辨率
- 训练数据可能包含大量高质量标注

---

## Gemini (Google, 闭源)

**特点：**

1. **原生多模态：** 从一开始就是多模态训练，不是后加的
2. **Ultra / Pro / Nano：** 多种尺寸
3. **视频理解：** 能理解长视频
4. **实时交互：** Gemini 1.5 支持实时多模态对话

**技术推测：**
- 使用 PaLM 系列架构
- 可能使用了 Perceiver 风格的 token 压缩
- 大规模多模态预训练

## 模型选择建议

| 需求 | 推荐模型 | 原因 |
|------|---------|------|
| **入门学习** | LLaVA-1.5 | 最简单，文档齐全 |
| **中文场景** | Qwen2-VL | 中文理解最好 |
| **高分辨率** | InternVL2 | 动态高分辨率支持 |
| **轻量部署** | Qwen2.5-VL-3B | 小模型高性能 |
| **视频理解** | Qwen2-VL / LLaVA-Video | 视频能力最强 |
| **文档/OCR** | InternVL2 | 文档理解最佳 |
| **追求性能** | GPT-4o / Gemini | 闭源最强 |
