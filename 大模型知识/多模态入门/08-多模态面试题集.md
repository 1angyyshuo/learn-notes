# 08 - 多模态面试题集

> 涵盖多模态模型核心知识点的面试题，分基础、进阶、高级、开放性四个难度等级。

---

## 基础题（20 题）

### 1. 什么是多模态学习？

**答：** 多模态学习是指让模型同时处理和理解多种类型的数据（文本、图像、视频、音频等），核心目标是让 AI 像人类一样综合多种感官信息进行理解和推理。

### 2. 多模态学习的三个核心阶段是什么？

**答：**
1. **Feature Extraction（特征提取）：** 从每种模态中提取有意义的特征
2. **Fusion（模态融合）：** 将不同模态的特征进行融合
3. **Generation（生成）：** 基于融合后的理解生成输出

### 3. ViT 是如何将图像转化为 token 序列的？

**答：**
1. 将图像切分为固定大小的 patches（如 16×16）
2. 每个 patch 展平为向量（16×16×3=768 维）
3. 通过线性投影映射到隐藏维度
4. 添加 [CLS] token 和位置编码
5. 输入 Transformer 编码器

### 4. CLIP 的训练目标是什么？损失函数怎么写？

**答：** CLIP 使用对比学习目标，拉近匹配的图文对，推远不匹配的。损失函数是 InfoNCE：

$$L = -\frac{1}{2N}\sum_{i=1}^{N}\left[\log\frac{e^{sim(I_i,T_i)/\tau}}{\sum_j e^{sim(I_i,T_j)/\tau}} + \log\frac{e^{sim(T_i,I_i)/\tau}}{\sum_j e^{sim(T_j,I_i)/\tau}}\right]$$

其中 sim 是余弦相似度，τ 是可学习温度参数。

### 5. LLaVA 的视觉 token 是如何注入 LLM 的？

**答：**
1. 图像经过 CLIP ViT 编码为视觉特征
2. 通过 MLP 投影层映射到 LLM 的文本嵌入空间
3. 将视觉 token 与文本 token 拼接
4. 整体输入 LLM 生成回答

### 6. 什么是早期融合、晚期融合和交叉注意力融合？

**答：**
- **早期融合：** 直接拼接原始特征，然后一起编码
- **晚期融合：** 各模态独立编码后融合
- **交叉注意力：** 一种模态的特征作为 Query，另一种作为 Key/Value

### 7. 为什么需要视觉语言对齐？

**答：** 视觉编码器和语言模型在不同的特征空间中训练。视觉特征编码颜色、形状等视觉信息，文本嵌入编码语义和语法。对齐的目的是将两者映射到同一语义空间，让 LLM 能"看懂"图像。

### 8. Q-Former 是什么？它解决了什么问题？

**答：** Q-Former 是一个使用可学习查询 token 的小型 Transformer，通过交叉注意力从视觉特征中提取固定数量（如 32 个）的 token。它解决了视觉 token 数量过多导致 LLM 计算量过大的问题。

### 9. 动态分辨率是什么？为什么重要？

**答：** 动态分辨率允许模型处理任意尺寸的图像，而不是固定为 224×224。这很重要因为：
- 高分辨率图像保留更多细节
- 不同宽高比的图像无需拉伸变形
- 适应各种实际场景

### 10. SigLIP 与 CLIP 有什么区别？

**答：** SigLIP 用 Sigmoid 损失替代了 CLIP 的 Softmax（InfoNCE）损失。优势是每个图文对独立计算，不需要全局 softmax，可以用更大的 batch size，训练效率更高。

### 11. 什么是幻觉（Hallucination）在多模态中？

**答：** 模型生成了与图像内容不符的描述。例如图像中只有一只猫，模型说"图中有一只猫和一只狗"。分为：
- 对象幻觉：描述了不存在的物体
- 属性幻觉：错误描述物体属性
- 关系幻觉：错误描述物体间关系

### 12. 多模态模型的三阶段训练是什么？

**答：**
1. **预训练：** 冻结视觉编码器和 LLM，只训练投影层，用大规模图文对
2. **SFT：** 指令微调，解冻部分或全部参数，用指令数据
3. **RLHF/DPO：** 偏好对齐，用偏好数据训练

### 13. ViT 的位置编码有什么作用？

**答：** Transformer 本身不包含位置信息，位置编码告诉模型每个 patch 在图像中的位置，保留空间关系。没有位置编码，模型无法区分左上角和右下角的内容。

### 14. 什么是 token 压缩？为什么需要？

**答：** Token 压缩是减少视觉 token 数量的方法（如 Q-Former、空间合并、Perceiver）。需要是因为视觉 token 数量直接影响 LLM 的计算量——attention 是 O(n²) 复杂度。

### 15. CLIP 的零样本能力是怎么来的？

**答：** CLIP 同时训练图像和文本编码器，使得匹配的图文对在嵌入空间中靠近。对于新类别，只需要用文本描述（如"a photo of a {class}"），无需重新训练就能分类。

### 16. 视觉编码器的 [CLS] token 有什么作用？

**答：** [CLS] token 是一个特殊的可学习向量，放在 patch 序列的开头。经过 Transformer 编码后，它的输出代表整个图像的全局语义信息，常用于分类任务。

### 17. 为什么 LLaVA-1.5 比 LLaVA 1.0 好很多？

**答：**
- 视觉分辨率从 224 提高到 336
- 投影层从线性层改为 2 层 MLP
- 训练数据从 150K 增加到 665K
- 更多的训练 epoch
- 这些简单改进带来了显著提升

### 18. 图像分片（Tiling）是什么？

**答：** 将高分辨率图像切分为多个小块（tiles），每个 tile 独立编码，再加上全局缩略图。这样可以在有限的 token 预算内处理高分辨率图像。

### 19. DINOv2 和 CLIP 的视觉特征有什么区别？

**答：**
- **CLIP：** 通过图文对比学习，特征偏向语义（"这是猫"）
- **DINOv2：** 通过自蒸馏学习，特征偏向视觉结构（形状、纹理）
- CLIP 需要文本配对，DINOv2 只需图像

### 20. 多模态模型中，为什么有时候要冻结视觉编码器？

**答：**
- 视觉编码器（如 CLIP ViT）已经学到了很好的视觉表示
- 解冻会增加训练参数和显存需求
- 解冻可能导致过拟合或灾难性遗忘
- 冻结可以加速训练，降低成本

## 进阶题（15 题）

### 21. Qwen2-VL 的动态分辨率是如何实现的？

**答：**
1. 根据图像尺寸动态调整 patch 数量（保持宽高比）
2. 设置最小/最大像素数限制
3. 使用 2D-RoPE 位置编码（分别编码高度和宽度位置）
4. 支持任意宽高比的图像输入，不需要 padding 或裁剪

### 22. Cross-attention 和 Visual Token 两种对齐方式各有什么优劣？

**答：**
| | Cross-Attention | Visual Token |
|---|---|---|
| 深度交互 | ✅ 每层都交互 | ❌ 只在输入时拼接 |
| 计算复杂度 | 高（每层都计算注意力） | 低（只拼接一次） |
| 修改 LLM | 需要修改每层结构 | 不修改 LLM |
| 训练成本 | 高 | 低 |
| 少样本学习 | 好（Flamingo 验证） | 一般 |
| 实现复杂度 | 高 | 低 |

### 23. 如何缓解多模态模型的幻觉问题？

**答：**
1. **数据层面：** 过滤幻觉标注，增加高质量数据
2. **训练层面：** 使用 DPO/RLHF，对幻觉回答施加负奖励
3. **架构层面：** 使用更大的视觉编码器，增加视觉 token 数量
4. **推理层面：** 设计验证机制，检测并修正幻觉
5. **评估层面：** 使用 POPE 等指标评估幻觉程度

### 24. Perceiver Resampler 和 Q-Former 有什么区别？

**答：**
- **Perceiver Resampler：** 固定数量的可学习查询 + 交叉注意力从视觉特征提取
- **Q-Former：** 更复杂的结构，包含自注意力+交叉注意力+FFN
- Q-Former 需要额外的预训练阶段
- Perceiver 更简单，Q-Former 表达能力更强

### 25. 多模态训练中，为什么有时要混入纯文本数据？

**答：** 防止灾难性遗忘。多模态训练可能破坏 LLM 原有的语言能力。混入纯文本数据让模型在学习视觉理解的同时保持语言能力。通常混合比例约 1:1 到 1:3。

### 26. 如何评估多模态模型的性能？

**答：** 常用 benchmark：
- **VQA-v2：** 视觉问答准确率
- **GQA：** 组合视觉问答
- **TextVQA：** 包含文字的视觉问答
- **POPE：** 幻觉评估
- **MMBench：** 综合能力评估
- **MMMU：** 多学科多模态理解
- **SEED-Bench：** 多维度评估

### 27. LoRA 在多模态训练中如何应用？

**答：** 通常只对 LLM 部分应用 LoRA，视觉编码器和投影层全参数训练。这是因为：
- 视觉编码器参数量相对较小
- 投影层需要全参数学习对齐
- LLM 参数量大，LoRA 节省显存
- LoRA 可以保留 LLM 的原始能力

### 28. 视频多模态和图像多模态的主要区别是什么？

**答：**
- **额外维度：** 视频多了时间维度，需要时序建模
- **计算量：** 视频帧数多，token 数量爆炸
- **采样策略：** 需要选择关键帧
- **时间关系：** 需要理解事件顺序和因果
- **音频轨道：** 视频通常包含音频

### 29. 如何设计一个好的视觉指令数据？

**答：**
1. **多样性：** 覆盖各种任务（描述、问答、推理、计数、OCR 等）
2. **复杂度梯度：** 从简单到复杂
3. **高质量：** 准确、详细、无幻觉
4. **开放性：** 鼓励详细回答，不是简单 yes/no
5. **grounding：** 回答基于图像内容

### 30. 为什么 LLaVA 用简单的 MLP 投影就能工作得很好？

**答：**
1. CLIP ViT 已经学到了高质量的语义表示
2. MLP 足以将 CLIP 特征空间映射到 LLM 空间
3. 复杂的投影层（如 Q-Former）反而可能过拟合
4. 大量的 SFT 数据弥补了简单架构的不足
5. 实验验证：简单的 MLP + 足够数据 > 复杂架构 + 少量数据

### 31. 2D-RoPE 和 1D-RoPE 有什么区别？为什么多模态需要 2D？

**答：**
- **1D-RoPE：** 只编码序列中的位置（第 1 个、第 2 个...）
- **2D-RoPE：** 分别编码高度和宽度位置
- 多模态需要 2D 因为图像本质是二维的，模型需要理解空间关系（如"左上角"、"右下角"）

### 32. 多模态模型的梯度检查点如何工作？

**答：** 在前向传播时只保存部分中间激活值，反向传播时重新计算其他激活。用计算时间换内存空间。对视觉编码器和 LLM 都可以启用。

### 33. 如何处理多模态训练中的模态不平衡问题？

**答：**
- **数据不平衡：** 图文数据远多于纯文本或纯图像
- **损失权重：** 对不同模态的损失设置不同权重
- **课程学习：** 先学简单任务，再学复杂任务
- **数据采样：** 对少数模态过采样

### 34. 什么是渐进式对齐训练（InternVL 的方法）？

**答：**
1. 先在大规模弱对齐数据上预训练视觉编码器
2. 用中等规模数据训练 MLP 投影层
3. 用高质量数据全模型微调
4. 每个阶段逐步增大模型和数据规模
5. 避免一次性训练大模型的不稳定性

### 35. 推理时如何优化多模态模型的速度？

**答：**
1. **KV Cache：** 缓存已计算的 Key/Value
2. **视觉 Token 压缩：** 减少输入 token 数量
3. **量化：** INT8/INT4 量化 LLM
4. **投机解码：** 用小模型快速生成，大模型验证
5. **批量推理：** 同时处理多个请求

## 高级题（10 题）

### 36. 多模态 RLHF 和纯文本 RLHF 有什么区别？

**答：**
| 方面 | 纯文本 RLHF | 多模态 RLHF |
|------|------------|------------|
| 奖励来源 | 文本质量 | 视觉 grounding + 文本质量 |
| 幻觉检测 | 较容易 | 需要对比图像内容 |
| 偏好标注 | 文本对比 | 需要视觉专家 |
| 奖励模型 | 纯文本 RM | 多模态 RM |
| 训练稳定性 | 相对稳定 | 更不稳定 |

### 37. 如何设计一个高效的视频理解模型？

**答：** 关键设计决策：
1. **采样策略：** 分层采样（全局+局部）
2. **时间建模：** SlowFast 设计（慢通道高分辨率 + 快通道低分辨率）
3. **Token 压缩：** 时间维度 + 空间维度双重压缩
4. **长视频处理：** 滑动窗口 + 层次化摘要
5. **多模态融合：** 视觉 + 音频 + 字幕
6. **训练策略：** 先在短视频上训练，再扩展到长视频

### 38. 请设计一个多模态模型的训练 pipeline

**答：**
```
阶段 0: 数据准备
  - 收集图文对（LAION-5B 过滤版）
  - 收集指令数据（GPT-4 生成）
  - 构造偏好数据

阶段 1: 视觉编码器预训练
  - CLIP 对比学习 或 DINOv2 自监督
  - 数据：大规模图文对
  - 产出：预训练 ViT

阶段 2: 投影层预训练
  - 冻结 ViT + LLM
  - 只训练 MLP 投影
  - 数据：图文描述
  - 产出：对齐后的投影层

阶段 3: SFT
  - 解冻部分参数
  - 数据：视觉指令
  - 产出：能遵循指令的模型

阶段 4: RLHF/DPO
  - 偏好对齐
  - 数据：偏好标注
  - 产出：最终模型

阶段 5: 评估
  - 基准测试（VQA, MMBench 等）
  - 幻觉评估（POPE）
  - 人工评估
```

### 39. 多模态模型的涌现能力是什么？如何解释？

**答：** 涌现能力是指模型在训练数据中没有显式教授的能力：
- **空间推理：** 理解物体的相对位置
- **属性绑定：** 将颜色与物体正确关联
- **计数：** 准确数出物体数量
- **OCR：** 读取图像中的文字

**解释：** 这些能力来自于视觉编码器和 LLM 的组合效应。CLIP 学到了丰富的视觉语义，LLM 学到了强大的推理能力，两者结合产生了新能力。

### 40. 如何解决多模态训练中的梯度冲突问题？

**答：** 不同任务/模态的梯度可能互相矛盾。解决方法：
1. **梯度投影：** 将梯度投影到一致方向
2. **任务权重：** 动态调整各任务的损失权重
3. **课程学习：** 从简单任务开始
4. **交替训练：** 不同任务交替训练
5. **多目标优化：** 使用 Pareto 最优

### 41. 多模态大模型的推理优化有哪些前沿方法？

**答：**
1. **Speculative Decoding：** 小模型快速生成 + 大模型验证
2. **KV Cache 压缩：** 减少缓存的 Key/Value
3. **视觉 Token 剪枝：** 移除不重要的视觉 token
4. **量化：** AWQ/GPTQ/SmoothQuant
5. **Flash Attention：** 优化 attention 计算
6. **Paged Attention：** vLLM 的高效内存管理

### 42. 如何构建一个高质量的多模态训练数据集？

**答：**
1. **来源多样化：** 互联网、专业领域、合成数据
2. **质量过滤：** CLIP 相似度过滤、去重、人工审核
3. **平衡性：** 各任务、各领域数据平衡
4. **难度梯度：** 从简单到复杂
5. **数据增强：** 改写、翻译、多视角
6. **安全过滤：** 移除有害、隐私内容

### 43. MoE（Mixture of Experts）如何在多模态中发挥作用？

**答：** 以 DeepSeek-VL2 为例：
- 不是所有专家都处理视觉信息
- 设有专门的视觉专家和文本专家
- 路由机制自动分配 token 到合适专家
- 好处：更高效，视觉和文本可以独立优化
- 挑战：路由不稳定，需要精心设计

### 44. 多模态模型的安全和伦理问题有哪些？

**答：**
1. **隐私：** 模型可能识别图像中的人脸、车牌
2. **偏见：** 训练数据中的偏见会被放大
3. **虚假信息：** 可能生成误导性的图像描述
4. **版权：** 训练数据可能包含版权内容
5. **滥用：** 可能被用于监控、跟踪等
6. **对策：** 数据过滤、安全对齐、使用限制

### 45. 未来多模态模型的发展方向是什么？

**答：**
1. **原生多模态：** 从一开始就是多模态训练，不是后加视觉
2. **实时交互：** 支持实时视频流、语音对话
3. **具身智能：** 多模态模型驱动机器人
4. **更长上下文：** 处理小时级视频、大量文档
5. **更高效：** 小模型实现大模型的能力
6. **多语言：** 更好的跨语言多模态理解
7. **Agent：** 多模态模型作为 Agent 的感知核心

## 开放性题（5 题）

### 46. 如果你有无限算力，如何设计下一代多模态模型？

**参考思路：**
- 原生多模态架构（从第一层就融合）
- 统一的 token 化方案（所有模态转为统一表示）
- 超大规模预训练（万亿参数，万亿数据）
- 端到端训练（不需要分阶段）
- 统一理解+生成（既能看也能生成）

### 47. 如何让多模态模型具有"常识"？

**参考思路：**
- 训练数据中包含更多日常场景
- 加入物理规律的监督信号
- 引入因果推理任务
- 与仿真环境交互获取经验
- 结合知识图谱

### 48. 多模态模型能否真正"理解"图像？

**参考思路：**
- 当前模型更多是统计关联，不是因果理解
- 涌现能力表明某种形式的"理解"
- 需要更严格的评估方法
- 可能需要具身经验才能真正理解

### 49. 如何设计一个让普通人也能训练多模态模型的框架？

**参考思路：**
- 简化 API（3 行代码训练）
- 自动超参数选择
- 内置数据清洗
- 一键部署
- 云端训练 + 本地推理

### 50. 多模态 RL 的独特挑战是什么？如何解决？

**参考思路：**
- 状态空间巨大（图像维度高）
- 奖励稀疏且难以定义
- 探索困难（图像空间太大）
- 解决：分层 RL、模仿学习、LLM 指导的奖励设计
