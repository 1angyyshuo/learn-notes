# 07 - 多模态模型训练方法

## 概述

多模态大模型的训练通常分为多个阶段，每个阶段有不同目标和数据需求。

---

## 三阶段训练范式

### 阶段一：预训练（Pre-training / Alignment）

**目标：** 让视觉编码器的输出与语言模型对齐

```
冻结视觉编码器 + 冻结 LLM
    ↓
只训练投影层（MLP/Q-Former）
    ↓
大规模图文对数据
```

**详细配置：**

| 配置项 | 典型值 | 说明 |
|--------|--------|------|
| 数据量 | 558K-5M 图文对 | LLaVA-1.5 用 558K |
| Batch Size | 256 | 取决于 GPU 数量 |
| 学习率 | 1e-3 ~ 1e-4 | 投影层用较大学习率 |
| Epoch | 1 | 通常 1 epoch 就够 |
| 优化器 | AdamW | 标准配置 |
| 冻结模块 | ViT + LLM | 只训练投影层 |

**数据格式：**
```json
{
    "image": "path/to/image.jpg",
    "conversations": [
        {"role": "user", "content": "描述这张图片"},
        {"role": "assistant", "content": "这是一张...的图片"}
    ]
}
```

**为什么只训练投影层？**
- 视觉编码器（CLIP ViT）已经学到了很好的视觉表示
- LLM 已经有了强大的语言能力
- 只需要学会"翻译"两者之间的语言

### 阶段二：SFT（Supervised Fine-Tuning）

**目标：** 让模型学会遵循视觉指令

```
解冻视觉编码器（可选）+ 投影层 + LLM
    ↓
视觉指令数据
    ↓
多任务学习
```

**详细配置：**

| 配置项 | 典型值 | 说明 |
|--------|--------|------|
| 数据量 | 150K-665K 指令 | LLaVA-1.5 用 665K |
| Batch Size | 128 | |
| 学习率 | 2e-5 ~ 5e-5 | 比预训练小 |
| Epoch | 1-3 | |
| ViT 是否解冻 | 可选 | 解冻效果更好但耗显存 |
| LLM | 全参数训练 | |

**数据构造：**

**方式一：GPT-4 生成（LLaVA 方法）**
```python
# 1. 给 GPT-4 提供图像的 caption 和 bounding box
prompt = f"""
图像描述: {caption}
物体标注: {bboxes}
请为这张图像生成 10 个高质量的问答对...
"""

# 2. GPT-4 生成多样化的指令数据
# - 简单描述："这张图里有什么？"
# - 复杂推理："为什么这个人看起来很高兴？"
# - 计数："图片中有几个人？"
# - OCR："图片中的文字是什么？"
```

**方式二：多任务数据混合**
```python
training_data = [
    ("图片描述", captioning_data),
    ("VQA", vqa_data),
    ("OCR", ocr_data),
    ("目标检测", detection_data),
    ("图像推理", reasoning_data),
]
```

### 阶段三：RLHF / DPO（偏好对齐）

**目标：** 让模型的输出更符合人类偏好

```
训练好的多模态模型
    ↓
构造偏好数据（好回答 vs 坏回答）
    ↓
DPO / RLHF 训练
    ↓
对齐后的模型
```

**偏好数据构造：**

```python
# 对于一张图像和一个问题
question = "图片中有几只猫？"
image = load_image("cats.jpg")

# 好回答（chosen）
good_answer = "图片中有 3 只猫。一只是橘猫在沙发上，一只是黑猫在窗台上，还有一只是白猫在地上。"

# 坏回答（rejected）
bad_answer = "有猫。"

# 或者：幻觉回答（chosen 应该是不幻觉的）
hallucinated_answer = "图片中有 2 只猫和 1 只狗。"  # 图中没有狗
correct_answer = "图片中有 3 只猫。"  # 正确
```

**DPO 损失函数：**

$$
\mathcal{L}_{DPO} = -\mathbb{E}\left[\log\sigma\left(\beta\log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta\log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]
$$

其中：
- $y_w$ 是好回答（chosen）
- $y_l$ 是坏回答（rejected）
- $\pi_\theta$ 是当前模型
- $\pi_{ref}$ 是参考模型（冻结）

**多模态 RLHF 的特殊挑战：**

| 挑战 | 说明 | 解决方案 |
|------|------|---------|
| 幻觉标注 | 需要标注模型是否产生幻觉 | 用 GPT-4V 辅助标注 |
| 视觉 grounding | 回答是否基于图像内容 | 引入 grounding 检测 |
| 偏好一致性 | 不同人对图像描述的偏好不同 | 多人标注 + 投票 |
| 训练不稳定性 | 多模态 RL 更容易不稳定 | 较小学习率 + 梯度裁剪 |

## 数据构造详解

### 图文对数据（预训练用）

**来源：**
- LAION-5B（50 亿图文对，互联网爬取）
- COYO-700M（7 亿图文对）
- CC3M/CC12M（Conceptual Captions）

**质量过滤：**
```python
def filter_image_text_pairs(pairs, min_clip_score=0.3):
    """用 CLIP 过滤低质量图文对"""
    filtered = []
    for image, text in pairs:
        # 计算 CLIP 相似度
        score = compute_clip_similarity(image, text)
        if score >= min_clip_score:
            filtered.append((image, text))
    return filtered
```

### 指令数据（SFT 用）

**来源：**
- LLaVA-Instruct-150K（GPT-4 生成）
- LLaVA-Instruct-665K（更大规模）
- 各种任务数据集（VQA、OCR、Captioning 等）

**数据增强：**
```python
def augment_instruction(question, answer, image):
    """指令数据增强"""
    # 1. 改写问题
    question_variants = [
        f"请描述这张图片: {question}",
        f"关于这张图片，{question}",
        question,
    ]
    
    # 2. 改写回答风格
    answer_variants = [
        f"根据图片，{answer}",
        answer,
        f"我看到图片中，{answer}",
    ]
    
    return list(zip(question_variants, answer_variants))
```

### 偏好数据（RLHF/DPO 用）

**构造方法：**
```python
def construct_preference_data(model, image, question):
    """为同一输入生成多个回答，构造偏好对"""
    # 生成多个回答
    answers = []
    for _ in range(4):
        answer = model.generate(image, question, temperature=0.7)
        answers.append(answer)
    
    # 用 GPT-4 或人工评分
    scores = [gpt4_score(image, question, a) for a in answers]
    
    # 构造偏好对
    best = answers[scores.index(max(scores))]
    worst = answers[scores.index(min(scores))]
    
    return {"chosen": best, "rejected": worst}
```

## 训练技巧

### LoRA 在多模态中的应用

```python
from peft import LoraConfig, get_peft_model

# 对 LLM 部分应用 LoRA
lora_config = LoraConfig(
    r=16,  # LoRA rank
    lora_alpha=32,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none"
)

# 只对 LLM 应用 LoRA，视觉编码器和投影层全参数训练
llm = get_peft_model(llm, lora_config)
```

**显存节省：**
- 全参数训练：~80GB（7B 模型）
- LoRA 训练：~24GB（7B 模型）

### DeepSpeed ZeRO

```python
# DeepSpeed ZeRO Stage 2 配置
ds_config = {
    "zero_optimization": {
        "stage": 2,
        "offload_optimizer": {"device": "cpu"},
        "allgather_partitions": True,
        "allgather_bucket_size": 2e8,
        "reduce_scatter": True,
        "reduce_bucket_size": 2e8
    },
    "fp16": {"enabled": True},
    "gradient_accumulation_steps": 4,
    "train_micro_batch_size_per_gpu": 4
}
```

### 梯度检查点（Gradient Checkpointing）

```python
# 启用梯度检查点，用时间换空间
model.vision_encoder.gradient_checkpointing_enable()
model.llm.gradient_checkpointing_enable()

# 梯度累积
for step, batch in enumerate(dataloader):
    loss = model(batch) / accumulation_steps
    loss.backward()
    
    if (step + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

### 混合精度训练

```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for batch in dataloader:
    with autocast():  # FP16 前向传播
        loss = model(batch)
    
    scaler.scale(loss).backward()  # 缩放 loss 防止下溢
    scaler.step(optimizer)
    scaler.update()
```

## 常见问题与调试

### 问题 1：Loss 不收敛

**可能原因：**
- 学习率太大或太小
- 数据质量问题
- 投影层维度不匹配

**解决方法：**
```python
# 1. 降低学习率
lr = 1e-5  # 从 1e-4 降到 1e-5

# 2. 检查数据
for batch in dataloader:
    print(batch['image'].shape, batch['text'].shape)
    # 确保维度匹配

# 3. 检查投影层
print(model.projection)
# Linear(in_features=1024, out_features=4096) ← 确保维度正确
```

### 问题 2：幻觉严重

**可能原因：**
- 训练数据中有幻觉标注
- 投影层过拟合
- 缺乏 RLHF 对齐

**解决方法：**
```python
# 1. 增加 RLHF/DPO 训练
# 2. 使用更大的视觉编码器
# 3. 数据过滤，移除幻觉标注
# 4. 增加 grounding 数据
```

### 问题 3：灾难性遗忘

**可能原因：**
- 多模态训练破坏了 LLM 的语言能力

**解决方法：**
```python
# 1. 在 SFT 数据中混入纯文本数据
sft_data = multimodal_data + text_only_data

# 2. 使用 LoRA 而不是全参数微调

# 3. 较小的学习率
```

### 问题 4：显存不够

**解决方法：**
```python
# 1. 梯度检查点
model.gradient_checkpointing_enable()

# 2. DeepSpeed ZeRO-3
# 3. 冻结视觉编码器
for param in model.vision_encoder.parameters():
    param.requires_grad = False

# 4. 减小 batch size + 增大梯度累积
batch_size = 1
accumulation_steps = 32

# 5. LoRA
from peft import get_peft_model
model = get_peft_model(model, lora_config)
```

## 代码示例：微调多模态模型

### 使用 transformers 微调 LLaVA

```python
from transformers import LlavaForConditionalGeneration, AutoProcessor
from peft import LoraConfig, get_peft_model
import torch

# 1. 加载模型
model_id = "llava-hf/llava-1.5-7b-hf"
model = LlavaForConditionalGeneration.from_pretrained(
    model_id,
    torch_dtype=torch.float16,
    device_map="auto"
)
processor = AutoProcessor.from_pretrained(model_id)

# 2. 配置 LoRA
lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: 4,194,304 || all params: 7,000,000,000 || trainable%: 0.06%

# 3. 准备数据
def prepare_data(examples):
    images = [Image.open(path) for path in examples["image_path"]]
    prompts = [f"USER: <image>\n{q}\nASSISTANT:" for q in examples["question"]]
    answers = examples["answer"]
    
    inputs = processor(
        text=prompts,
        images=images,
        return_tensors="pt",
        padding=True
    )
    
    labels = processor.tokenizer(answers, return_tensors="pt", padding=True).input_ids
    inputs["labels"] = labels
    return inputs

# 4. 训练
from transformers import TrainingArguments, Trainer

training_args = TrainingArguments(
    output_dir="./llava-finetuned",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-5,
    fp16=True,
    save_steps=500,
    logging_steps=10,
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
    data_collator=prepare_data,
)

trainer.train()
```
