# 05 - 视频多模态理解

## 概述

视频理解 = 多帧图像理解 + 时间维度建模。比图像多模态多了时间序列的挑战。

---

## 核心挑战

| 挑战 | 说明 | 解决思路 |
|------|------|---------|
| **计算量** | 1 分钟视频 = 1800 帧 (30fps)，token 数爆炸 | 采样策略、Token 压缩 |
| **时间关系** | 理解事件发生的顺序和因果 | 时间建模 |
| **长视频** | 电影、直播等长时内容 | 滑动窗口、摘要 |
| **细粒度** | 快速动作、表情变化 | 关键帧检测 |
| **多模态** | 视频 = 视觉 + 音频 + 字幕 | 多模态融合 |

## 视频采样策略

### 1. 均匀采样（Uniform Sampling）

最简单的方法，等间隔取帧：

```python
def uniform_sampling(frames, num_samples=32):
    """从视频帧中均匀采样"""
    indices = np.linspace(0, len(frames)-1, num_samples, dtype=int)
    return [frames[i] for i in indices]
```

**优点：** 简单，覆盖整个视频
**缺点：** 可能漏掉关键帧

### 2. 关键帧采样（Keyframe Sampling）

检测场景变化，采样关键帧：

```python
def keyframe_sampling(frames, threshold=30):
    """基于帧差检测关键帧"""
    keyframes = [frames[0]]
    for i in range(1, len(frames)):
        diff = np.mean(np.abs(frames[i] - frames[i-1]))
        if diff > threshold:
            keyframes.append(frames[i])
    return keyframes
```

**优点：** 保留重要信息
**缺点：** 可能错过缓慢变化

### 3. 动态采样（Dynamic Sampling）

根据视频内容动态调整采样密度：

```python
def dynamic_sampling(frames, min_fps=1, max_fps=4):
    """
    静态场景低采样率，动态场景高采样率
    """
    scene_complexity = compute_complexity(frames)  # 计算场景复杂度
    target_fps = min_fps + (max_fps - min_fps) * scene_complexity
    return sample_at_fps(frames, target_fps)
```

### 4. 分层采样（Hierarchical Sampling）

先粗采样定位时间段，再细采样：

```
原始视频 30fps × 60s = 1800 帧
    ↓
第一层：均匀采样 64 帧（全局概览）
    ↓
第二层：在关键时间段采样更多帧（细节）
```

## 时间建模方法

### 方法一：帧级拼接（Frame Concatenation）

最简单的方法——将所有帧的特征拼接成序列：

```python
def frame_concatenation(frames, encoder):
    """每帧独立编码，然后拼接"""
    features = []
    for frame in frames:
        feat = encoder(frame)  # (1, 256, 1024)
        features.append(feat)
    # 拼接成序列
    sequence = torch.cat(features, dim=1)  # (1, 32*256, 1024)
    return sequence
```

**问题：** Token 数量 = 帧数 × 每帧 patch 数，32 帧 × 256 patch = 8192 tokens

### 方法二：时序注意力（Temporal Attention）

加入专门的时间注意力层：

```python
class TemporalAttention(nn.Module):
    def __init__(self, hidden_dim, num_heads=8):
        super().__init__()
        self.temporal_attn = nn.MultiheadAttention(hidden_dim, num_heads)
        self.norm = nn.LayerNorm(hidden_dim)
    
    def forward(self, frame_features):
        """
        frame_features: (num_frames, num_patches, hidden_dim)
        """
        # 跨帧时间注意力
        # 每个 patch 关注其他帧中相同位置的 patch
        T, N, D = frame_features.shape
        
        # 重塑为 (patches, frames, dim) 以跨帧计算注意力
        x = frame_features.permute(1, 0, 2)  # (N, T, D)
        
        # 时间注意力
        attn_out = self.temporal_attn(x, x, x)
        x = self.norm(x + attn_out)
        
        return x.permute(1, 0, 2)  # (T, N, D)
```

### 方法三：3D 卷积

用 3D 卷积同时捕捉空间和时间特征：

```python
class Conv3DEncoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv3d = nn.Conv3d(
            in_channels=3,
            out_channels=64,
            kernel_size=(3, 8, 8),  # (时间, 高, 宽)
            stride=(1, 8, 8)
        )
    
    def forward(self, video):
        """
        video: (batch, channels, num_frames, H, W)
        """
        features = self.conv3d(video)  # (batch, 64, T, H//8, W//8)
        return features.flatten(2).transpose(1, 2)  # (batch, T*H//8*W//8, 64)
```

### 方法四：光流（Optical Flow）

计算相邻帧之间的像素运动：

```
帧 t ────► 光流计算 ────► 运动场 (u, v)
帧 t+1
```

**优点：** 显式建模运动信息
**缺点：** 计算量大，需要额外光流估计

## 视频 Token 化策略

### 问题

原始视频的 token 数量太大：
- 1 分钟视频 @ 1fps = 60 帧 × 256 tokens = 15,360 tokens
- 1 分钟视频 @ 4fps = 240 帧 × 256 tokens = 61,440 tokens

### 压缩策略

**1. 时间维度压缩：** 每 N 帧合并为一个 token

```python
def temporal_compression(features, factor=2):
    """
    每 2 帧合并为 1 个 token
    (T, N, D) → (T//2, N, D)
    """
    T, N, D = features.shape
    features = features.reshape(T//factor, factor, N, D)
    features = features.mean(dim=1)  # 时间维度平均
    return features  # (T//2, N, D)
```

**2. 空间维度压缩：** 合并相邻 patches

```python
def spatial_compression(features, factor=2):
    """
    2x2 patches 合并为 1 个 token
    """
    T, N, D = features.shape
    h = w = int(N**0.5)
    features = features.reshape(T, h, w, D)
    
    # 合并
    merged = features.reshape(T, h//factor, factor, w//factor, factor, D)
    merged = merged.mean(dim=(2, 4))  # 空间维度平均
    
    return merged.reshape(T, -1, D)  # (T, N//4, D)
```

**3. Qwen2-VL 的方案：** 每 2 帧合并 + 动态分辨率

## 主流视频多模态模型

### Video-LLaVA

```
视频帧 → LanguageBind（视觉+语言绑定编码器）→ 拼接 → Vicuna
```

**特点：** 使用预训练的 LanguageBind 编码器，统一图像和视频编码

### Qwen2-VL 视频模式

```
视频帧（动态采样）
    ↓
每 2 帧合并为一组
    ↓
ViT 编码（每组独立）
    ↓
MLP 投影
    ↓
Qwen2 LLM
```

**特点：**
- 原生支持视频，无需特殊处理
- 支持长视频（数小时）
- 2D-RoPE 位置编码自然处理时间维度

### LLaVA-Video (2024)

```
视频帧
    ↓
均匀采样 64 帧
    ↓
每帧：CLIP ViT → 576 特征
    ↓
SlowFast 设计：
  - Slow pathway: 少量帧，高空间分辨率
  - Fast pathway: 多帧，低空间分辨率
    ↓
拼接 → LLaVA-Video LLM
```

**SlowFast 设计：**
```python
class SlowFastVideoEncoder:
    def encode(self, video, slow_fps=1, fast_fps=4):
        # Slow: 高分辨率，少帧
        slow_frames = sample(video, slow_fps)
        slow_features = [self.encoder(frame, high_res=True) for frame in slow_frames]
        
        # Fast: 低分辨率，多帧
        fast_frames = sample(video, fast_fps)
        fast_features = [self.encoder(frame, low_res=True) for frame in fast_frames]
        
        return torch.cat(slow_features + fast_features, dim=1)
```

## 长视频处理挑战与解决方案

### 挑战

| 视频长度 | 帧数 (1fps) | Token 数 (256/帧) | 问题 |
|---------|------------|-------------------|------|
| 10 秒 | 10 | 2,560 | 可处理 |
| 1 分钟 | 60 | 15,360 | 勉强 |
| 10 分钟 | 600 | 153,600 | 超出上下文 |
| 1 小时 | 3,600 | 921,600 | 不可能 |

### 解决方案

**1. 滑动窗口**

```python
def sliding_window(video, window_size=32, stride=16):
    """分段处理长视频"""
    segments = []
    for i in range(0, len(video), stride):
        segment = video[i:i+window_size]
        if len(segment) < window_size:
            break
        segments.append(segment)
    return segments
```

**2. 层次化摘要**

```
长视频 → 分段 → 每段生成摘要 → 汇总摘要 → 最终理解
```

**3. 检索增强**

```
视频 → 索引（帧特征 + 时间戳）
用户提问 → 检索相关帧 → 只处理相关帧
```

**4. 视频压缩**

- 减小帧分辨率
- 增大采样间隔
- 只保留关键帧
